# AZ104 Projects x5

## Project 1: Azure Compute and Identity Management

**Topics Covered**: VMs, RBAC, Azure AD, Policies, Encryption, Cost Management

**Time**: ~0.75 hours

### Summary

This project involves deploying a virtual machine, securing it using RBAC, applying Azure Policies for resource governance, encrypting sensitive data, and monitoring costs. It teaches foundational concepts for managing Azure identities, governance, and compute resources.

### Scenario

A company wants to deploy a virtual machine for their web application, secure it with role-based access control, enforce a policy for naming conventions, encrypt sensitive data, and monitor the cost of the deployed resources.

### Steps

#### 1. Create a Virtual Machine

- Go to **Azure Portal** > **Virtual Machines** > **Create**.
- Select subscription, resource group, region, VM size, and OS (Windows/Linux).
- Configure an admin username/password or SSH key.
- Add a **data disk** during creation:
  - Create a new disk > None.
  - *Note:* Selecting Storage Blob allows you to reuse disks from outside Azure or custom configurations you uploaded.
- Configure **Networking**:
  - Assign a static private IP via the **Advanced Networking** section.
  - Use a **Network Security Group (NSG)** to allow or block traffic:
    - **None:** Use if you're managing the NIC setup manually or attaching an existing NIC.
    - **Basic:** Use for quick and simple deployments, such as test environments or small-scale applications.
    - **Advanced:** Use when you need fine-grained control (e.g., production workloads or custom network setups).
- Review and create the VM.
- Download private key.

#### 2. Configure Role-Based Access Control (RBAC)

- **Create a Key Vault and store the private key** (apply least privilege principle):
  - Go to **Key Vault** > **Create** > Use the same resource group and region (required) > Review + create.
  - Go to **Entra ID** > **Create a group** > Select yourself as owner and member.
  - Go to **Key Vault** > **IAM** > **Add role assignment** > Role: **Key Vault Administrator** > Users: Select your new group > Review + assign (propagation may take a few minutes).
    - **Sign out and sign back in** to Azure Portal to refresh your token.
  - Go to **Key Vault** > **Objects** > **Keys** > **Generate/Import** > **Import** > Upload the private key file.
- Navigate to the **VM** > **Access Control (IAM)** > **Add Role Assignment**.
- Assign the **"Virtual Machine Contributor"** role (or a custom role) to a user or group.
- Verify permissions:
  - Go to **Entra ID** > **Manage** > **Groups** > **All groups** > Select your group name.

#### 3. Log into the VM

- Go to **VM** > Select your VM > **Connect** > **SSH using Azure CLI** > **Configure**.
- In the new CLI window that opens, run:

  ```bash
  az network public-ip list

Test the public IP in a browser to confirm it is only accessible via SSH (not HTTPS) due to NSG rules.

#### 4. Apply Azure Policy

- Navigate to Azure Policy > Authoring > Definitions > Initiative definition.

- Enter Name > Select Category > Click Next.

- Add policy definition(s):

  - Search for tag → Select:

    - "Inherit a tag from resource group…"

    - "Inherit a tag from subscription_name"

- Search for allow → Select "Allowed resource types".

- Click Review + Create > Next.

- Create a group > Tags > Save > Previous.

- Select tag policies > Add to a group > Select Tags > Save.

- Select Initiative parameters > Create initiative parameter > Enter Name, DisplayName, and Allowed Values > Save.

- For reference IDs (Inherit a tag from resource group… and Inherit a tag from subscription_name):

  - Set Value Type = Use Initiative Parameter

  - Set Value(s) = Name

  - Set Allowed resource types = virtualMachines

- Click Review and create > Create.

- Assign the initiative to the subscription and test:

  - Select the subscription under Basics.

  - Configure parameters under Parameters.

  - Click Review + create > Create.

- To view policy and compliance: Go to VM > Operations > Policies.

#### 5. Monitor and Manage Costs

- Navigate to Cost Management + Billing > Select your Billing scope > Cost management > Budgets > Add:

  - Set a monthly spending limit for the subscription.

  - Configure email or action group notifications for thresholds (e.g., 50%, 80%, 100%).

- Review Cost Analysis to identify spending trends and optimize costs.

- Review Advisor recommendations for cost-saving suggestions.
